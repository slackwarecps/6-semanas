A team enables the Pull Request Action in its repository. What behavior is automatically expected on each pull request that is created?
[Resposta correta: A Pull Request Action revisa as mudanças, analisa o impacto e posta um relatório no PR. Ela não faz merge automático (A), não se limita a rodar testes (C) e publica o retorno no próprio PR, não por email (D).]

---
[ ] A - Automatically merges the PR if no conflict is detected.
[ ] B - Reviews the proposed changes, analyzes the impact and posts a detailed report on the PR.
[ ] C - Only runs the test suite and fails the PR in case of failure.
[ ] D - Notifies the author by email without publishing anything on the PR itself.
