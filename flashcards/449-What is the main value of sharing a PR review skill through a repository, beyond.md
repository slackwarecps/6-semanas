What is the main value of sharing a PR review skill through a repository, beyond individual use?
[Resposta correta: Compartilhar a skill via repositório padroniza o code review em todo o time e cria uma experiência consistente na organização, não só para quem a criou. Não se trata de cache ou custo de API (A), não elimina configuração de hooks (C) e o objetivo é justamente ampliar o uso além do autor, não restringir a ele (D).]

---
[ ] A - Reduces the API cost per review by reusing cache across users.
[ ] B - Standardizes code review across the whole team and creates a consistent experience in the organization.
[ ] C - Eliminates the need for any hook configuration in the repository.
[ ] D - Ensures that only the skill author keeps control over the reviews.
