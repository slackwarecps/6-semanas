When is it preferable to use few-shot examples instead of refining instructions to obtain a consistent output format?
[Resposta correta: Few-shot é preferível depois que o refinamento de instruções já falhou, pois 3-4 exemplos do formato exato são seguidos com mais confiabilidade que instruções abstratas. Não é uma escolha automática antes de tentar instruções (A), few-shot aumenta tokens (C) e mais contexto explicativo não corrige variabilidade de formato (D).]

---
[ ] A - Always, even before trying to refine the instructions.
[ ] B - When refining the instructions has already failed; 3-4 examples showing the exact format are followed more reliably than abstract instructions.
[ ] C - When you want to reduce the number of tokens in the prompt.
[ ] D - When it is enough to add more explanatory context to the prompt.
