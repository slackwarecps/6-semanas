How do you organize a large set of project instructions in a modular and maintainable way?
[Resposta correta: Arquivos por tópico em .claude/rules/ dão modularidade e manutenção. Um CLAUDE.md gigante não é modular; README é documentação humana, não instrução para o Claude; hooks são por evento, não organizam diretrizes.]

---
[ ] A - Put everything in a single giant CLAUDE.md
[ ] B - Create separate markdown files by topic in .claude/rules/ (e.g. testing.md, api-conventions.md)
[ ] C - Distribute the instructions across several README.md files in the repo
[ ] D - Create one hook per instruction topic
