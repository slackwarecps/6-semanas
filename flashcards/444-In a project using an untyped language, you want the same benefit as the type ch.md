In a project using an untyped language, you want the same benefit as the type checking hook. What is the equivalent approach?
[Resposta correta: Sem compilador ou type checker, o equivalente é usar testes automatizados para capturar regressões e devolver o feedback ao Claude. Forçar tipos estáticos (B) foge da linguagem escolhida; desativar hooks (C) abre mão do benefício; um linter de estilo (D) verifica formatação, não a correção funcional que os testes cobrem.]

---
[ ] A - Implement similar functionality using automated tests in place of the type checker.
[ ] B - Force the adoption of static type annotations in the untyped language.
[ ] C - Disable hooks, since type checking does not apply to untyped languages.
[ ] D - Run a style linter as a direct replacement for the type checker.
