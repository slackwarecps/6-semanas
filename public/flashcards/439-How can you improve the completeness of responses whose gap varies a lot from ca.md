How can you improve the completeness of responses whose gap varies a lot from case to case?
[Resposta correta: Um passo de autocrítica (padrão evaluator-optimizer) faz o agente avaliar o rascunho contra critérios antes de responder, tratando lacunas variáveis. Few-shot serve a padrões consistentes, não lacunas altamente variáveis (A), trocar o modelo não resolve gap estrutural (B) e pedir confirmação ao cliente transfere o ônus (D).]

---
[ ] A - Add few-shot examples covering every possible variation.
[ ] B - Swap the model for a larger one.
[ ] C - Add a self-critique step in which the agent evaluates its own draft against criteria before responding.
[ ] D - Ask the customer to confirm whether the response turned out complete.
