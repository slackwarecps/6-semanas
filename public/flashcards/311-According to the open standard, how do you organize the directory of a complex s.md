According to the open standard, how do you organize the directory of a complex skill?
[Resposta correta: O open standard sugere scripts/, references/ e assets/ para separar código, documentação e dados. A A ignora a organização; C e D descrevem estruturas de projeto ou de hooks, não de skill.]

---
[ ] A - Everything in a single SKILL.md, without subdirectories
[ ] B - scripts/ for executable code, references/ for documentation and assets/ for images/templates/data
[ ] C - src/, dist/ and node_modules/
[ ] D - One directory per hook event
