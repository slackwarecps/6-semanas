What is the recommended good practice when naming XML tags in a prompt?
[Resposta correta: Nomes específicos e descritivos ajudam Claude a entender o propósito de cada seção. Nomes genéricos (A), aleatórios (C) ou numéricos (D) não comunicam a função da seção e reduzem a clareza.]

---
[ ] A - Always use generic, short names like <a> or <x> to save tokens.
[ ] B - Use specific, descriptive names like <sales_records>, <my_code> or <docs>.
[ ] C - Use random names so as not to induce biases in the model.
[ ] D - Use only numeric tags like <1> and <2>.
