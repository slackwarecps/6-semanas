Why is providing the existing content in the context the reliable way to avoid duplication, such as duplicate tests?
[Resposta correta: O LLM só raciocina sobre o que está em seu contexto, então ver o conteúdo existente é o que permite evitar duplicatas. O matching por palavra-chave é frágil e perde equivalências semânticas (A), reduzir a saída não trata duplicação (C) e o modelo não memoriza o repositório entre execuções (D).]

---
[ ] A - Because keyword matching detects all duplicates robustly.
[ ] B - Because the LLM only reasons over what is in its context; without seeing what already exists, there is no way to avoid duplicates.
[ ] C - Because reducing the amount of output eliminates duplicates.
[ ] D - Because the model memorizes the repository between runs.
