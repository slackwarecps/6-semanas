For which types of output is the prefill plus stop sequence technique appropriate?
[Resposta correta: A técnica serve para qualquer conteúdo formatado onde se quer só o conteúdo sem explicações, incluindo Python, bullets e CSV. Ela não é exclusiva de JSON (A), não se destina a texto explicativo longo (B) e não depende de extended thinking (D), que inclusive é incompatível com prefilling.]

---
[ ] A - Only for JSON, since it depends on the curly-brace syntax.
[ ] B - Only for long natural-language responses that need context.
[ ] C - For any structured content without comments, such as Python snippets, bulleted lists or CSV.
[ ] D - Only when extended thinking is enabled.
