Saying that MCP is "transport agnostic" means that:
[Resposta correta: Transport agnostic quer dizer flexibilidade de transporte, com stdio local sendo o caso comum. A assume HTTP como único. B ignora o cenário local via stdio. D descreve uma automação que não existe.]

---
[ ] A - Communication only works via HTTP between different machines.
[ ] B - Client and server always run on separate remote servers.
[ ] C - Client and server can communicate through different methods (stdio, HTTP, WebSockets), with stdin/stdout on the same machine being the most common.
[ ] D - The protocol chooses the transport on its own without the developer being able to configure it.
